# Color Clusterer

Десктопное приложение для анализа доминирующих цветов изображения и генерации компактной цветовой палитры.

Программа считывает изображения форматов **PNG**, **JPEG** и **BMP**, выделяет **6** основных цветов с помощью алгоритма **k-средних** в цветовом пространстве **CIELAB (Lab)** и визуализирует результат рядом с исходным изображением.

---

# Возможности

## Анализ изображений

- Загрузка форматов PNG, JPEG и BMP через стандартный диалог выбора файлов
- Преобразование изображения в RGBA
- Выборка пикселей с адаптивным шагом для ускорения обработки изображений высокого разрешения

## Генерация палитры

- Кластеризация цветов методом k-средних в цветовом пространстве Lab
- Инициализация центроидов алгоритмом **k-means++**
- Формирование палитры из **6** доминирующих цветов
- Сортировка цветов по размеру кластера (от наиболее распространенного к наименее)

## Визуализация

- Отображение исходного изображения на белом фоне
- Вывод палитры в виде шести цветных квадратов с HEX-кодами
- Автоматическая загрузка демонстрационного изображения при запуске

## Обработка прозрачности

- Удаление альфа-канала с корректной обработкой полупрозрачных пикселей (альфа-блендинг с белым фоном)

# Алгоритмическое обеспечение

## Преобразование RGB => Lab

### 1. Линеаризация sRGB

```text
R' = R / 12.92                     , если R <= 0.04045
R' = ((R + 0.055) / 1.055)^2.4     , если R > 0.04045
```

### 2. Преобразование в CIE XYZ

```text
|X|   |0.4124 0.3576 0.1805|   |R'|
|Y| = |0.2126 0.7152 0.0722| x |G'|
|Z|   |0.0193 0.1192 0.9505|   |B'|
```

### 3. Преобразование XYZ → Lab

Белая точка D65:

```text
Xn = 95.047
Yn = 100.000
Zn = 108.883
```

Функция:

```text
f(t) = t^(1/3),                    если t > (6/29)^3
f(t) = (1/3)(29/6)^2t + 4/29,      иначе
```

Формулы:

```text
L = 116 × f(Y/Yn) - 16
a = 500 × (f(X/Xn) - f(Y/Yn))
b = 200 × (f(Y/Yn) - f(Z/Zn))
```

## Обработка прозрачности

Полупрозрачные пиксели смешиваются с белым фоном:

```text
Rрез = α × Rисх + (1 - α) × Rфон
```

где:

```text
Rфон = 255
```

Аналогично вычисляются каналы **G** и **B**.

Полностью прозрачные пиксели (`α = 0`) исключаются из кластеризации.

---

## Выборка пикселей

```text
step = ceil(sqrt(pixelsCount / maxCount))
```

где

```text
maxCount = 600000
```

В итоге для изображений, который содержат более 600 000 пикселей, будет обработано от 150 000 до 600 000 пикселей.

## Алгоритм k-means++

Инициализация центроидов:

1. Первый центроид выбирается случайным образом.
2. Для каждой точки вычисляется квадрат расстояния до ближайшего центроида.
3. Новый центроид выбирается с вероятностью, пропорциональной квадрату расстояния.
4. Процесс повторяется до выбора всех `k` центроидов.

---

## Алгоритм k-средних

Основной цикл:

1. Назначение каждой точки ближайшему центроиду.
2. Пересчет центроидов.
3. Проверка смещения (`eps`).
4. Замена пустых кластеров случайными точками.

Завершение происходит при отсутствии изменений центроидов либо при достижении максимального числа итераций.

### Метрика расстояния

```text
ΔE = sqrt((L2 - L1)^2 + (a2 - a1)^2 + (b2 - b1)^2)
```

---

## Обратная конвертация

После кластеризации центры кластеров преобразуются:

```text
Lab => XYZ => RGB => HEX
```

# Сборка и запуск

## Требования

- Компилятор с поддержкой C++17
- Qt 6.x
- CMake 3.16+

## Сборка через CMake

```bash
# конфигурация
cmake -B build -S .

# сборка
cmake --build build

# запуск
./build/clusterer
```

# Использование

1. При запуске автоматически загружается пример изображения.
2. Нажать кнопку **«Выбрать файл»**.
3. Выбрать изображение формата PNG, JPEG или BMP.
4. Получить палитру из шести доминирующих цветов с HEX-кодами.
